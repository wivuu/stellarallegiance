# Iron Coalition thin tech tree ‚Äî import plan

## Context

The live content bundle (`server/Content/core/`) is placeholder-grade: four made-up guns (scout/fighter/bomber/heavy-cannon), two runtime bases, a 4-node stub research tree, and a nameless stock faction. The mission (from `.PLAN/TechPathsFromMemory.md`) is to adopt a **thin slice of Allegiance's Iron Coalition faction** ‚Äî decoded from `/Users/erik/projects/Allegiance/artwork-full/PCore014.igc` via the `igc-format` skill ‚Äî so the game inherits a real, battle-tested tech tree's stats and feel: ships, gun lines with tiered research, base upgrade chains, and faction-level stat multipliers. Mechanics we lack (ripcord/teleport, cloak, boarding) are deferred.

The tech pipeline (YAML ‚Üí projection ‚Üí MsgDefs wire ‚Üí DefRegistry ‚Üí ResearchTab/per-base research queue) is fully built and data-driven; unlock-only tech gating is pure YAML. The genuinely new code is: **stat-multiplier plumbing** (library `AttributeResolver` exists but is dropped at projection, absent from the wire, never applied in sim), **station upgrades** (`Station.SuccessorStationId` exists but is unprojected/unused), **per-base-type research slots + max health** (both hardcoded to type 0 today ‚Äî `World.cs:364/988/1006` all use the single garrison `BaseMaxHealth`), and a **heal path** for the Nanite gun.

## Adaptation decisions

1. **Research re-homing** (IGC homes Mini-Gun/Nanite tiers at the Expansion base, which the slice lacks): tier 2 at Garrison (`required-capabilities: [base]`), tier 3 behind `garrison-str` tech (granted by Upgrade Garrison). Gat/AutoCan 2 require `supremacy-1` (granted when Supremacy is built ‚Äî station completion already grants techs, `Simulation.Constructors.cs:932-940`), Gat/AutoCan 3 require `supremacy-adv`. Prefer techs over the closed Capability enum for all new gates.
2. **Upgrade mechanic = YAML-scoped** (user decision 2026-07-16): developments that trigger a station upgrade carry a new field `upgrade-scope: all | single` (default `all`).
   - `all`: completing the dev upgrades every live matching base of the team, and matching bases completed later spawn pre-upgraded.
   - `single`: only the base that HOSTED the research upgrades (the research engine is already per-base ‚Äî `MsgResearch` carries `baseId`, state in `ResearchByBase`, so `CompleteResearch` knows the hosting site). Other bases of the same type stay un-upgraded; later-built bases spawn at the base tier. Garrison‚ÜíGarrison (Str) and any future Outpost upgrade are authored `single`; Supremacy/Shipyard upgrades authored `single` too for consistency (each team owns ‚â§1 anyway).
   - One-shot for now: the dev is owned once; upgrading a second base of the same type would need dev repeatability ‚Äî out of scope, noted as future work.
   - Guard: a `single`-scope upgrade dev must be researched AT a base of the from-type (validate at `ResearchOpStart` ‚Äî reject queuing it elsewhere with a notice), else the completion has no valid target.
   - Tech grants stay team-wide either way; only the physical base swap is scoped.
3. **Stat translation = keep our magnitudes, import Allegiance ratios.** Anchors: current `fighter` (speed 100, hp 120, mass 36) ‚Üî IGC Enh Fighter for hulls; current fighter-cannon DPS (10 dmg/4 ticks) ‚Üî PW Gat Gun 1 for guns. `fire-interval-ticks = round(dtimeReady√ó20)`; projectile power set to preserve IGC DPS ratios; `projectile-life-ticks` capped near current 16 (our world is smaller). Prices: outpost 300 ‚Üî IGC 5000 (√ó0.06) ‚Üí Supremacy 600, Shipyard 900, Upgrade Garrison 720, Upgrade Supremacy 1800, Upgrade Heavy Class 1440. Document anchor ratios as comments atop `hulls.yaml`/`weapons.yaml`.
4. **Inert-but-authored attributes**: `max-energy` √ó1.20 (no ship energy model exists) and `max-shield-station` √ó1.15 (bases have no shields) are authored for fidelity with a one-line comment; no new mechanics invented.
5. **Nanite heal = explicit `is-healing` flag** on the weapon (not negative power): heals hull only, clamps to `MaxHull`, bypasses shields, friendly targets only; zero effect on enemies. Client tints bolt/hit feedback off the flag. Exclude `IsHealing` guns from PIG threat/`PrimaryWeapon` selection.
6. **Cleanup**: delete placeholder guns/bolts/techs (`heavy-ordnance`, `cannon-tier-2`, `expansion-1`, `tactical-1`), their devs, and the five catalog-only placeholder stations (refinery/tech-lab/supremacy-center/expansion-complex/teleport-receiver). BuildTab renders from the streamed catalog ‚Äî self-updating. Outpost keeps `base-type-id: 1` + `expansion-allowed` grant. Launcher weapon-ids 3‚Äì8 stay untouched (dispenser "unequipped" sentinel is weapon-id 0). Maps only place garrisons ‚Äî untouched.
