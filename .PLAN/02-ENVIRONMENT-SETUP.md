# 02 — Environment Setup

## Authoritative references (prefer these over memory)

APIs in this stack shift between releases. When unsure, consult:

- SpacetimeDB docs: https://spacetimedb.com/docs
- SpacetimeDB C# quickstart: https://spacetimedb.com/docs/quickstarts/c-sharp
- SpacetimeDB C# client SDK reference: https://spacetimedb.com/docs/clients/c-sharp
- SpacetimeDB tables: https://spacetimedb.com/docs/tables
- SpacetimeDB functions (reducers): https://spacetimedb.com/docs/functions
- SpacetimeDB subscriptions: https://spacetimedb.com/docs/clients/subscriptions
- Blackholio (multiplayer asteroids in Godot) tutorial — closest reference implementation
  to this prototype; mirror its replication/prediction patterns where possible.
- Godot C# docs: https://docs.godotengine.org/en/stable/tutorials/scripting/c_sharp/

> The prototype targets **SpacetimeDB 2.0** (the `[SpacetimeDB.Table]` /
> `[SpacetimeDB.Reducer]` attribute API with `ReducerContext`). If the installed CLI reports
> a 1.x version, stop and record it in `99` — the 1.x → 2.0 API differs and this spec assumes 2.0.

## Required toolchain

| Tool | Version | Notes |
|------|---------|-------|
| SpacetimeDB CLI (`spacetime`) | 2.0.x | `spacetime --version` to confirm. |
| .NET SDK | 8.0 | Required by both the module and the Godot C# client. |
| .NET WASI workload | experimental | `dotnet workload install wasi-experimental` — module compiles to WASM. |
| Godot | 4.x **.NET / Mono build** | Must be the C#-capable build, not the GDScript-only build. |
| Git | any recent | Version control. |

## Install steps (run once)

```bash
# 1. Install the SpacetimeDB CLI (see https://spacetimedb.com/install for your OS)
#    then authenticate (GitHub-backed):
spacetime login

# 2. Install the WASI workload for compiling C# modules to WebAssembly:
dotnet workload install wasi-experimental

# 3. Confirm versions:
spacetime --version      # expect 2.0.x
dotnet --version         # expect 8.0.x
```

Godot 4 (.NET build) is installed separately from its website; it is not a CLI package.

## Project bootstrap commands

```bash
# Create the module (this also wires up a starter; we will replace Lib.cs contents per doc 03/04)
spacetime dev --template basic-cs    # scaffolds module + local server + generates bindings

# Generate / regenerate C# client bindings into the Godot project after schema changes:
spacetime generate --lang csharp --out-dir client/module_bindings --project-path module

# Run the local standalone server (foreground; use a dedicated terminal):
spacetime start

# Publish the module to the local server (name the DB 'stellar-allegiance'):
spacetime publish --project-path module stellar-allegiance

# Tail logs:
spacetime logs stellar-allegiance
```

## Godot ↔ SpacetimeDB wiring

1. Add the SpacetimeDB C# client SDK to the Godot C# project (NuGet package referenced from
   the client `.csproj`; confirm the current package name/version from the C# client SDK
   reference link above).
2. Place generated bindings in `client/module_bindings/` and ensure the `.csproj` includes them.
3. `ConnectionManager.cs` opens the connection to `ws://localhost:3000` (local) for the
   database named `stellar-allegiance`, registers subscription queries, and exposes the
   connection to the rest of the scene tree (autoload singleton recommended).

## Definition of "environment ready"

Environment setup is complete when the unmodified `basic-cs` template publishes to the local
server, the `client` Godot project compiles in the .NET build of Godot, and a placeholder
connection from Godot to the local DB succeeds (logged "connected"). This is the acceptance
test for Task 0 in `08`.
